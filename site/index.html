Two jobs, done safely

Chronoframe always shows you a plan before it changes anything.

📁

Organize

Copy scattered media into folders like 2024/06/15. Chronoframe resolves each file's date from metadata, filename, and filesystem — and lets you review uncertain dates before anything is copied. Your source folder stays read-only.

🔍

Deduplicate

Find exact copies by content, not filename, plus near-duplicates, bursts, RAW+JPEG pairs, and Live Photos. You choose what to keep. Selected files move to the macOS Trash — never a hard delete.

Safety first, by design

The whole point of Chronoframe is that you can trust it with an irreplaceable library.

  • ✓ Originals stay untouchedChronoframe reads your source folder but never moves, renames, edits, or deletes source files.
  • ✓ You approve the planOrganize shows what will copy before transfer. Deduplicate shows what moves to Trash before commit.
  • ✓ No overwritesIf a destination filename already exists, Chronoframe creates a distinct name instead of replacing it.
  • ✓ Copies are verifiedTransfers are written atomically and re-hashed by default to confirm every copy is intact.
  • ✓ Trash, not hard deleteDeduplicate sends selected files to the macOS Trash so you can recover them.
  • ✓ Receipts and revertHistory records what happened, and supported runs can be reverted when the files still match the receipt.

How it works

From a messy folder to an organized library in a few clear steps.

Choose your folders. Pick the messy source and a separate destination, then a folder layout.
Preview the plan. Chronoframe scans, resolves dates, and shows exactly what will copy.
Review what needs attention. Fix unknown or low-confidence dates before you commit.
Transfer, then clean up. Copy into place, then deduplicate to Trash when you're ready.
